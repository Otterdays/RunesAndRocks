## ⚡ The Engine Foundation

**Runes & Rocks** isn't just a game; it's a completely bespoke, bare-metal authoritative MMORPG engine built from the ground up to support massive multiplayer loads seamlessly across **Desktop** and **Android**.

- 🛡️ **Server-Authoritative:** The client is a strictly visual "dumb terminal." 100% of physics, collisions, and state calculations happen on the secure headless server.
- ⚙️ **Entity Component System (ECS):** Built with a robust, data-oriented custom Kotlin ECS engine processing highly efficient matrices. 
- ⏱️ **Fixed-Timestep Simulation:** Deterministic physical gameplay safely executing precisely 20 times a second independent of background thread IO lag.
- 📡 **High-Speed Networking:** Powered by raw TCP `Ktor` asymmetric socket multiplexing.
- 🗺️ **Godot-Ready Map Parsing:** Server accepts standard universal JSON 2D arrays (like ones generated by Godot or Tiled) and dynamically creates physical boundaries mathematically in milliseconds.

## 🏗️ Architecture Stack

Our strict multi-module philosophy ensures maximum decoupling. The `client` rendering codebase never leaks into the `server` simulation workspace. 

| Tech | Purpose |
| ---- | ------- |
| **Kotlin (JDK 17+)** | Zero-boilerplate core engine logic. |
| **Gradle (Kotlin DSL)** | Multi-module workspace abstraction (`:server`, `:client`, `:shared`, `:android`). |
| **Ktor (TCP)** | Asynchronous coroutine raw socket processing. |
| **LibGDX** | Native 60-FPS OpenGL hardware-accelerated map/sprite rendering on Windows & Mobile. |
| **KryoNet** | High-speed binary packet compilation. |
| **PostgreSQL & Redis** | Safe, transactional persistent economies and hot caching. |

## 🚀 Launching the Realm

The engine is currently highly portable for local development.

### 1. Launch the Authoritative Server
The server manages the heavy lifting. Double-click the root batch script to spin up the Ktor environment, lock in the 20 TPS `TickLoop`, and start streaming ECS physical matrix arrays:
> `launch_backend.bat` 

*Admin bonus: Visit `http://localhost:8080/` to monitor live server TPS, active socket clients, and raw memory loads.*

### 2. Enter the World (Desktop)
Fire up the LibGDX client to visually interpret incoming server data and traverse the map JSON layouts:
> `launch_client.bat` 

*(You can open as many `launch_client` windows side-by-side as your machine can handle to see real-time player syncing!)*

### 3. Enter the World (Android/Mobile)
To package an `.apk` native build for Android Studio:
1. Open `settings.gradle.kts`.
2. Delete the `//` before `include("android")`.
3. Open the project inside Android Studio.
4. Click Play!
